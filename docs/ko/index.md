# RAG 앱 체크리스트

RAG 앱을 만드셨군요. 로컬에서는 잘 돌아갑니다. 링크를 공유하시기 전에, 아래 두 가이드를 꼭 확인해 주세요.

---

## [망신당하지 않고 RAG 앱 배포하기](ship.md)

프로덕션에서 문제가 될 수 있는 부분들에 대한 복붙 해결책입니다:

- **환각 감지** — RAG가 지어내고 있는지 10줄로 확인하는 방법
- **안전 필터** — 프롬프트 인젝션과 개인정보 유출을 차단하는 함수 하나
- **비용 추적** — 채팅 한 번에 $0.08이 나가고 있다는 걸 청구서 전에 파악하기
- **검색 디버깅** — 왜 엉뚱한 청크가 돌아오는지, 어떻게 고치는지
- **청킹** — 실제로 잘 작동하는 설정값과 검증 방법
- **스트리밍** — 사용자가 5초간 스피너만 보는 상황을 방지하기
- **에러 처리** — LLM API는 반드시 실패합니다, 미리 대비해 두세요

## [RAG 앱 테스트하는 법](test.md)

LLM에는 `assert answer == "정확한 문자열"`을 쓸 수 없습니다. 대신 이렇게 하시면 됩니다:

- **검색 테스트** — 올바른 컨텍스트가 검색되는지 확인 (결정적, 무료)
- **생성 테스트** — 답변이 컨텍스트에 근거하는지, 환각인지 확인 (RAGAS)
- **안전 테스트** — 프롬프트 인젝션과 개인정보 처리를 parametrize로 검증
- **에이전트 테스트** — 도구 선택 정확도, 스텝 효율성, 환각률 측정
- **LLM 목킹** — 테스트 비용을 절약하는 방법
