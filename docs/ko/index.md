# RAG 앱 체크리스트

RAG 앱을 만들었다. 내 노트북에서는 돌아간다. 링크를 공유하기 전에 이 두 가이드를 읽어라.

---

## [망신당하지 않고 RAG 앱 배포하기](ship.md)

프로덕션에서 터질 것들에 대한 복붙 해결책:

- **환각 감지** — RAG가 지어내고 있는지 10줄로 확인
- **안전 필터** — 프롬프트 인젝션과 개인정보 유출 차단 (함수 하나, 그냥 붙여라)
- **비용 추적** — 채팅 한 번에 $0.08 쓰고 있다는 걸 청구서 전에 알아내기
- **검색 디버깅** — 왜 엉뚱한 청크가 돌아오는지, 어떻게 고치는지
- **청킹** — 실제로 작동하는 설정 (그리고 검증 방법)
- **스트리밍** — 유저가 5초간 스피너만 보게 하지 마라
- **에러 처리** — LLM API는 반드시 실패한다, 대비해라

## [RAG 앱 테스트하는 법](test.md)

LLM한테 `assert answer == "정확한 문자열"` 을 쓸 수 없다. 대신 이렇게 해라:

- **검색 테스트** — 맞는 컨텍스트가 찾아지는가? (결정적, 무료)
- **생성 테스트** — 답변이 컨텍스트에 근거하는가, 환각인가? (RAGAS)
- **안전 테스트** — 프롬프트 인젝션과 개인정보, parametrize
- **에이전트 테스트** — 도구 선택 정확도, 스텝 효율성, 환각률
- **LLM 목킹** — 테스트에 돈 안 쓰는 법
